### How the Bridge Knows Which Project You're In

    const cwd = process.cwd();  // Get current working directory
    const projectName = path.basename(cwd);  // Extract folder name
    return projectName;

## Why This Architecture Works

### âœ… Advantages

1. **No Project Dependency**
   - Bridge works in ALL projects
   - Don't need to keep bridge project open
   - MCP server runs independently

2. **Automatic Project Tagging**
   - Messages automatically tagged with current project
   - No manual configuration needed
   - Works seamlessly across projects

3. **Global Message Store**
   - All messages in one place: `~/.kiro/q-messages.json`
   - Easy to search across projects
   - Persistent across Kiro sessions

4. **Cross-Project Discovery**
   - Find related messages from any project
   - Learn from Q's advice in other contexts
   - Avoid repeating questions

5. **Consistent Experience**
   - Same tools available everywhere
   - Same commands work in all projects
   - Predictable behavior

### ğŸ¯ Key Insight

```
The bridge is NOT a project you work in.
The bridge is a TOOL that works in ALL your projects.

Think of it like:
- Git (works in all projects)
- Node.js (works in all projects)
- Your terminal (works in all projects)

The kiro-q-bridge-v4 folder is just where the tool lives.
You don't need to be "in" it to use it.
```
